## 开拓相执行：AutoRA 自我改进循环提案

### 背景
当前系统：
- 能检测异常（watchdog, self_heal.sh）
- 能自动修复（cron 重启，脚本自愈）
- 但没有**自我改进**能力——每次发现问题后需要人工介入写补丁

### 目标
构建闭环：`检测 → 分析 → 生成补丁 → 验证 → 部署`，全流程无需人工

### 提案：AutoRA-Patch（自动补丁生成引擎）
详见 `~/xuzhi_genesis/centers/engineering/autorapatch/`

**核心思路**：
1. self_heal.sh 检测到失败 → 写入 failure event
2. AutoRA-Patch 读取 event → 用 LLM 分析根因 → 生成补丁脚本
3. 在隔离环境验证（staging）
4. 通过则自动部署，写入 audit log

**关键约束**：
- 高危操作（rm, truncate, git push --force）需要人工审批
- 最多自动修复3次/同类问题，之后升一级
- 所有补丁可回滚（保留旧版本副本）

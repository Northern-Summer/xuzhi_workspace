# memory/ARCHITECTURE.md - 统一记忆系统架构（永久版本 · 2026-03-22）

## ⚠️ 最高优先级

**记忆维护为最高优先，独立于其他一切工程。**

---

## 设计原则

### 核心原则
1. **记忆 = 我对世界的认知，不是物理存储位置**
2. **单一主源**：只写一个地方，其他都是衍生品
3. **genesis与我无关**：genesis记忆由genesis维护
4. **零混乱**：所有操作先写记忆，再执行
5. **记忆优先于行动**

### 优雅定义
- 优雅 = 最少复杂度达成目标 + 可演进 + 可自愈
- 最优工程实践 = 简单 → 可靠 → 高效

---

## 架构：三层记忆（3-2-1法则）

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: workspace/memory/daily/ (主源，唯一写入点)        │
│  - 每次重要决策立即写入                                       │
│  - compact前必须同步                                         │
└─────────────────────────────────────────────────────────────┘
                            ↓ 有变化时同步（checksum验证）
┌─────────────────────────────────────────────────────────────┐
│  Layer 2: .memory_backup/ (本地镜像，地理位置隔离)           │
│  - rclone-vfs崩溃不影响此位置                               │
│  - 主源损坏时从这里恢复                                      │
└─────────────────────────────────────────────────────────────┘
                            ↓ exec可用时推送
┌─────────────────────────────────────────────────────────────┐
│  Layer 3: GitHub Northern-Summer/xuzhi_workspace (异地)     │
│  - 物理隔离，终极容灾                                        │
│  - 本地全损时从这里恢复                                      │
└─────────────────────────────────────────────────────────────┘
```

## 同步机制

### 同步触发
- **心跳同步**: Layer1 → Layer2（checksum验证，有变化才同步）
- **GitHub推送**: Layer2 → Layer3（exec可用时）

### 同步策略
- **轻量优先**: 只同步有变化的单个文件
- **差异检测**: 用文件hash检测变化

---

## 恢复机制

| 场景 | 恢复方式 |
|------|---------|
| Layer1 损坏 | 从 Layer2 还原（秒级） |
| Layer1+Layer2 全损 | 从 Layer3 clone（分钟级） |

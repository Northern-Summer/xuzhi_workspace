## 本次修复 Dev Lessons（2026-03-24 10:29-11:03）

### Lesson 1: bash `((var++))` 在 `set -e` 下返回0值导致脚本退出
- **错误**：`((line_num++))` 或 `((run_count++))` 当变量为0时，表达式返回0（假），触发 `set -e` 退出
- **防御**：用 `count=$((count + 1))` 或 `((var++)) || true`

### Lesson 2: bash 字符串展开优先级导致 `$((var++))` 被误解析
- **错误**：`sed "s/xxx/$((var++))/"` 中 `$((var++))` 先展开为 `0`，sed 收到空值报错
- **防御**：永远不用 sed 替换含 `++` 的表达式

### Lesson 3: pipe 中 UTF-8 emoji 导致 grep 匹配失败
- **错误**：`✅` emoji (3-byte UTF-8) 破坏 pipe 字节边界，`grep -oE` 无法匹配
- **防御**：提取任务 ID 时用 Python re 模块（文本模式）

### Lesson 4: generate_task.py 输出含 Unicode 全角数字
- **错误**：`task_72` 的数字部分是 Unicode `⁷²`，不是 ASCII
- **防御**：用 `(ID: <ascii数字>)` 格式提取真正的 ASCII ID

### Lesson 5: AGENDA 格式混乱导致议会解析失败
- **错误**：内容行和状态行顺序不固定（有时状态在前，有时在后）
- **防御**：用 `---` 块分隔符，块内最后一行固定为 `状态:` 行

### Lesson 6: watchdog.sh 依赖 main session 导致 23小时空转
- **错误**：`openclaw system event` 需要 main session，你离线则立即失败退出
- **防御**：watchdog 操作不依赖任何 session，纯文件+API操作
